## 📬 About

Institutional web platform developed for **Universidad Autónoma de Chiriquí (UNACHI)**. The system provides students with a formal channel to submit complaints, suggestions, and feedback — while giving administrators a structured workflow to manage, respond to, and resolve each case.

Built with security in mind: OWASP Top 10 principles were applied throughout development to protect sensitive student data.

> 🎓 University-scale deployment — handling real institutional data.

---

## ✨ Features

| Feature | Description |
|---|---|
| 📝 **Complaint Submission** | Students submit complaints with category, description, and supporting files |
| 🔄 **Lifecycle Tracking** | Full status tracking from submission → review → resolution |
| 👥 **Role-Based Access** | Separate interfaces for students and administrators |
| 📊 **Admin Dashboard** | Overview of all cases, filters, and status management |
| 🔔 **Notifications** | Status updates delivered to students on case progress |
| 🔐 **Secure by Design** | OWASP-aligned: input validation, XSS prevention, auth controls |

## 🔐 Security Practices Applied

| Area | Implementation |
|---|---|
| **Injection** | Prepared statements, query builder — no raw SQL |
| **XSS** | Output escaping on all user-generated content |
| **Authentication** | Session-based auth with role enforcement |
| **Access Control** | Route-level middleware for role verification |
| **Input Validation** | Server-side validation on all forms |

## 📬 Descripción

Plataforma web institucional desarrollada para la **Universidad Autónoma de Chiriquí (UNACHI)**. El sistema proporciona a los estudiantes un canal formal para enviar quejas, sugerencias y comentarios — mientras da a los administradores un flujo estructurado para gestionar, responder y resolver cada caso.

Construido con seguridad en mente: principios OWASP Top 10 aplicados en todo el desarrollo para proteger datos estudiantiles sensibles.

> 🎓 Despliegue a escala universitaria — manejando datos institucionales reales.

---

## ✨ Características

| Característica | Descripción |
|---|---|
| 📝 **Envío de Quejas** | Estudiantes envían quejas con categoría, descripción y archivos adjuntos |
| 🔄 **Seguimiento del Ciclo** | Seguimiento completo: envío → revisión → resolución |
| 👥 **Acceso por Roles** | Interfaces separadas para estudiantes y administradores |
| 📊 **Panel Administrativo** | Vista general de todos los casos, filtros y gestión de estados |
| 🔔 **Notificaciones** | Actualizaciones de estado entregadas a los estudiantes |
| 🔐 **Diseño Seguro** | Alineado con OWASP: validación de entradas, prevención XSS, controles de auth |

|---|---|

| **XSS** | Escapado de salida en todo contenido generado por usuarios |
